A Brief Description

In this Project we have implemented Google Chrome's offline game, well known as Dinosaur game using ATMega-32. The game is displayed on 8x16 LED matrix and played with joystick and sound sensor.

We have added an extra feature that our Dino can shoot the obstacles.





Required Equipments

  • ATmega32 microcontroller

  • LED matrix and 3x8, 4x16 Demultiplexer for muxing them

  • 5V Power Supply

  • Joystick

  • Sound sensor

  • Lcd module

  • Some other gates





How it works?

In this project ATmega32 microcontroller is used for interfacing to various hardware peripherals. A 8x16 LED display is made by multiplexing two 8x8 bicolor LED matrix. For shorting the rows of the two matrices, we have used a 3x8 decoder and for column-select, we have used 4x16 demultiplexer. For choosing the color, another 4x16 demultiplexer has been used.

For taking the input from user, a joystick and a sound sensor have been intrefaced with the microcontroller. When the joystick is moved right, the resistance increases over a certain value and our dino jumps. Our dino can duck in the same way when the resistance goes below a certain value.

Dino can shoot an obstacle based on the amplitude of the sound taken by the sound sensor.

Finally the score is displayed in a 16x2 Lcd module.

Algorithm



       1.Start
       2.Obstacle starts coming.
              3.If joystick is moved right, dino jumps.
              4.Else if joystick is moved left, dino ducks.
                            5.If the dino and the obstacle collides, go to (11).
                            6.Else increase score.
                                       7.Take input from the sound sensor and compare. If amplitude > 15000 then shoot the bird.
                                          8.If the dino and the bird collides, go to (11).
                                          9.Else increase score.
                                     10.Go to (2).
                              11.Game over.
                              12.Show score on Lcd display.
        13.End

Problems Faced

  1. The most diffiult problem we had to solve is to manage the data received through the sound sensor. It receives a lot of data in a very short interval and the value was fluctuating. It was difficult to sort out only the necessary attributes in a very short time.

  2. The resistance caused by the movement of the joystick had been changed after few trials. So we had to change our threshold resistance in our code.

  3. We faced problems with the ADC input. While taking multiple inputs from ADC, one channel influenced the inputs of other channels.
